# Glossary

## Pattern
A reusable workflow structure that describes the shape of agentic work independent of any one company, tool, or industry.

## Pattern family
A higher-level branch that groups related patterns such as `monitor-detect-triage` or `plan-coordinate-schedule`.

## Domain
A context where a pattern appears, such as finance, engineering, compliance, operations, research, support, or HR.

## Instance
A grounded Markdown example that shows how a canonical pattern appears in a specific real workflow or target environment.

## Vocabulary
A controlled YAML list of approved terms for concepts like domains, architectures, capabilities, autonomy levels, and risk levels.

## View
A navigation-oriented YAML artifact that reorganizes patterns for browsing, for example by domain, architecture, autonomy, or risk.

## Schema
The canonical machine-readable definition of what fields a pattern entry must or may contain.

## Execution memory
The committed `.agent/` files that preserve mission, plan, decisions, status, coverage, and iteration history across repository-building loops.

## Coverage matrix
A structured record of which combinations of pattern families, domains, architectures, and risk levels are currently represented.

## Phase 0
The bootstrap phase that creates the persistent execution-memory files under `.agent/`.

## Phase 1
The next phase, focused on foundational docs and the canonical pattern schema.
