# D-0016 — Define Phase 8 as breadth expansion after coverage refinement

- Status: accepted
- Decision: After Phase 7 coverage refinement, use Phase 8
  for controlled breadth expansion across patterns,
  newly justified domains, and their linked grounded
  instances. Reserve the following phase for deeper
  architecture, governance, and consistency work after
  that breadth expansion lands.
- Rationale: The repository has already closed the original
  family, domain, architecture, and risk coverage gaps
  inside the current modeled surface, so the next major
  phase should describe how to widen the ontology safely
  rather than only deepen the current matrix. Making
  Phase 8 explicitly about pattern, domain, and instance
  breadth keeps the dependency order intact, avoids
  architecture-first drift, and gives future iterations a
  clear rule for when new domains or canonical patterns
  are justified.
